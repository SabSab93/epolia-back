# Issue Planner

Use this skill before coding an issue that touches several files or has unclear scope.

## Workflow

1. Read the issue title, body, acceptance criteria, and out-of-scope notes.
2. Identify the smallest deliverable that satisfies the issue.
3. List files likely to change.
4. Call out risks, missing context, and validation commands.
5. Keep the plan short and avoid implementation details until coding starts.

## Output

- Scope: what is included.
- Excluded: what must not be changed.
- Files: expected files.
- Validation: commands to run.
